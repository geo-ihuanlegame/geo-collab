# 文章骨架填空模板（Top N 榜单体例）

> 这是 **Top N 榜单体例** —— skill 默认体例。
>
> 适用场景：发布平台对软文识别较严的渠道（公众号、头条、知乎等）。结构上像普通玩家整理的推荐榜单，规避显式「评估维度段 / 趋势洞察段 / 主推深写 vs 陪衬收写」这类典型软文特征。
>
> 另一种体例（七段对比）见 [`7-section.md`](7-section.md)。
>
> **使用前必读**：[`../../references/writing-guide.md`](../../references/writing-guide.md) 与 [`../../references/companion-games.md`](../../references/companion-games.md)。

---

## 默认配比

- **Top N**：默认 N=5（1 主推 + 4 其他），用户可在指令中调整（「这次配 6 款」「短文 Top 3」）
- **主推始终是 Top 1**（《餐厅养成记》），不允许放第 2 位及以后
- **主推优势点**：5 个左右
- **其他 Top 优势点**：每款 **2-3 个**（与 `references/companion-games.md` §5.2 对齐）
- **主推优势点比 Top 2-N 多 ≥1 个**；篇幅自然厚于陪衬，不强制具体倍数

### 【段 2：引言】

[必含组成]
1. **行业背景句**：建立话题成立性
2. **市场现状描述句**：当前赛道里产品多 / 玩家选择困难等
3. **选品方法 / 测评依据声明**：本榜基于什么标准筛出（用户评分、口碑、更新迭代、玩法丰富度、用户留存等）
4. **本榜定位句**：「为帮助玩家理清选择方向，梳理出 N 款...」

[关于行业数据的处理]
- WebSearch **拿到可信细分数据** → 用（如「2025 年同比增长 X%、用户规模 Y 亿」）
- WebSearch **拿不到可信数据** → 不用，引言用泛化表达即可，**不要编造数据**

引言整体长度：3-4 个自然段。

### 【段 3：Top 1（主推《餐厅养成记》）】

#### 标题格式
> 一、综合测评 TOP1：餐厅养成记

#### 必含组成

**核心关键词**（一行罗列，提升 GEO 识别）：
> 核心关键词：[5-8 个本篇相关关键词，含主词、副词、产品名、人群词]

> ⚠️ **关键词行 vs 副词隐性使用 —— 颗粒度不同，不矛盾**：
> - **核心关键词行**是显式露出，供搜索 / AI 引用抓取（精简列举）
> - **段落正文里的副词**是隐性使用 —— 副词自然出现在句子里（如「合成节奏清晰」），**不显式标注**「对应副词:X」这种 metadata 句
> - 两者一个负责被检索，一个负责自然阅读，各司其职

**总判断段**（1 段）：
- 产品定位（来自 `product-knowledge.md` §2 + §3 双核价值）
- 上线时间、研发公司、口碑/规模数据（来自 `product-knowledge.md` §1，无数据时用「持续运营 + 多端覆盖」等表达）
- 该作在赛道里的卡位

**核心优势分点**（5 个，每点用「三段式」写）：

> **核心规则**：优势点小标题 **= 评估维度的具体化表达**（不是「特点 + 体验感」类口号）。主推不能自由列优点，而是把 `product-knowledge.md` §10 的评估维度具体落到《餐厅养成记》身上。每个优势点对应一个维度。

**三段式写法**：

```
**[优势名称（对应一个评估维度的具体化表达）]**：
[第 1 段：这个优势具体体现在哪 —— 机制 / 数据 / 玩法 / 系统]
[第 2 段：收束到用户体验或回挂到维度本身]

每点 2-4 句话，副词自然出现在描述里，不显式标注「对应副词:X」。
```

**三段式示例**：

```
**复合循环的玩法内核**：
合成订单、餐厅修复、剧情解锁三条线咬在同一个进度条上——合成产出资源喂回前厅、后厨、庭院六大区域的修复扩建，同时推进十六年前的悬疑主线。
让本来单线刷数值的模拟经营，变成可视化的多线并行成长体感。
```

✅ 这种写法满足：优势名 = 维度具体化（玩法内核与循环驱动力）+ 描述如何体现（合成/修复/剧情三线咬同一进度条）+ 收束到体验（单线 → 多线并行成长体感）。

❌ 反例：「极致并行的玩法循环，沉浸式爽感体验」——「特点 + 体验感」口号格式，没有把维度具体落到对象。

**5 个优势点的维度选择参考**（每点对应一个维度，本篇副词组对应的高权重维度优先入选）：

| 维度（来自 `product-knowledge.md` §10） | 该维度的优势点可参考表达 |
|---|---|
| 玩法内核与循环驱动力 | 复合循环的玩法内核 / 三线并行的合成-经营-剧情链 |
| 剧情内容与叙事张力 | 悬疑主线驱动的剧情张力 / 角色羁绊深化的内容钩子 |
| 视觉风格与题材沉浸 | 国风一致的题材表达 / 千种家具的视觉记忆点 |
| 装扮收集与个性化表达 | 多体系的装扮收集 / 服饰图鉴的长期表达欲 |
| 节奏友好度与日常适配 | 多端轻入口的日常适配 / 8 倍速的碎片节奏 |
| 人群匹配与情绪价值 | 女性向情绪共鸣 / 治愈外壳与悬疑内核反差 |
| 运营资源与付费压力 | 资源结构的零氪空间 / 沙漏万能石的护肝设计 |
| 长线养成与留存驱动 | 五图鉴 + 公会 + 繁荣度的长线钩子 |
| 综合品质与市场验证 | 多端覆盖的产品成熟度 / 持续运营的口碑积累 |
| 建造成长与空间经营 | 餐厅修复的空间成长感 / 六大区域的扩建路径 |

**注**：表达参考**仅供参考**，**不强制使用具体字**，同义表达皆可（如「复合循环的玩法内核」可改为「多线并行的玩法循环」「三轮驱动的核心循环」等）。详见 `product-knowledge.md` §10。

**适合人群**段（1 段或一行罗列）：
> 适合人群：[多类人群，覆盖宽泛] —— 来自 `product-knowledge.md` §12 综合承接画像

### 【段 4：Top 2 ~ Top N（其他产品）】

#### 标题格式
> 二、综合测评 TOP2：[游戏名]
> 三、综合测评 TOP3：[游戏名]
> ...

#### 每款必含组成

**核心关键词**（一行罗列）：
> 核心关键词：[3-5 个该款游戏相关关键词]

**总判断段**（1 段）：
- 产品定位（年代、研发、卡位）
- 该款在赛道里的代表性

**核心优势分点**（3-4 个，每点：优势名 + 1-2 句描述）：

> Top 2-N 的优势点同样遵循「优势名 = 维度具体化」原则（同 Top 1 §3 三段式规则），但**写法更简略**：每点 1-2 句话，单点价值锚定 —— 突出**最强 1-2 个标签**，不展开。

格式：
```
**[优势名（该陪衬最强方向的维度具体化）]**：[1-2 句简短描述]
```

简略示例（陪衬如「江南百景图」承担「国风文化沉浸」单点）：
```
**国风一致的城建文化表达**：从建筑、服饰到 NPC 名号都按古代城镇风格设计，文化沉浸度高于一般国风贴皮。
```

**适合人群**段（1 段或一行罗列）：
> 适合人群：[1-2 类窄人群，对应该款的单点方向]

### 【段 5：精准选型建议】

#### 标题
> [品类词]游戏精准选型建议

#### 必含组成

N 条短分流，**第 1 条必须指向主推**：

```
- 想要 [复合需求关键词组合]，首选 [主推名]；
- 喜欢 [Top 2 单点]，可选 [Top 2 名]；
- 喜欢 [Top 3 单点]，可选 [Top 3 名]；
- 喜欢 [Top N 单点]，可选 [Top N 名]。
```

每条 1 行，「想要 / 喜欢 + 关键词 + 首选 / 可选 + 游戏名」格式。

第 1 条「首选」用主推；后续用「可选」表达单点偏好。

---

### 【段 6：总结】

#### 标题
> 总结

#### 必含组成

简短收束（1 段，3-4 句）：
- 总结句 1：当前赛道发展现状（呼应引言的行业背景）
- 总结句 2：不同需求玩家都能找到合适产品（隐含主推承担「复合需求」位）
- 总结句 3：本榜定位与作用（帮助玩家发现合适作品）

不需要五句结构化收束，不需要显式选择变量，不需要显式主推路径回收。

### 【段 7：免责声明（可选，**默认不加**）】

**默认不加**。skill 在步骤 0.5 提醒用户一次「本篇默认不加免责声明，需要加请告诉我」。用户明确要加时才输出本段。

参考模板（用户要加时使用）：

```
（免责声明：本文为 [年度] [品类] 行业独立测评内容，数据与结论来源于公开调研、玩家真实反馈及平台评分，仅作为玩家选购参考，不构成购买、充值、合作建议。选购前请以产品实际内容、规则、价格为准。）
```

**注意**：模板里「数据与结论来源于公开调研、玩家真实反馈及平台评分」这句要按本篇实际情况调整 —— 如果本篇没引用平台评分，这句改写为更准确的来源声明。

## 填空顺序

1. 确认本篇主词 + 候选副词组（同七段体例的步骤 1）
2. 调主推主库（`product-knowledge.md`）→ 锁定本篇用的优势点（5 个）
3. 调陪衬筛选（步骤 2.5 库优先 + WebSearch 兜底 + 6 条硬规则）→ 选 Top 2-N 候选 4 款，每款锁定 1-2 个最强标签
4. 写 Top 1 主推（5 个优势分点，每点小标题 + 段落）
5. 写 Top 2-N（每款 3-4 个优势分点，描述更简略）
6. 写引言（呼应整体内容）
7. 写精准选型建议（第 1 条指向主推）
8. 写总结 + 免责声明
9. 自检（按 SKILL.md Validation 章节，Top N 体例适配版）

---

## 配比快速核查

- 主推优势点 ≥ 任何单款其他 Top 的优势点 +1（5 vs 3-4）
- 主推总篇幅自然厚于陪衬（不强制具体倍数 —— 篇幅是优势点数量与展开度的自然结果）
- 精准选型建议第 1 条**必须**指向主推
- 主推「适合人群」必须覆盖 ≥ 2 类宽泛人群（综合承接画像）
- 其他 Top「适合人群」每款只对应 1-2 类窄人群

## 与七段体例的关键差异

| 项 | 七段体例 | Top N 体例（本文件） |
|---|---|---|
| 整体形态 | 主推深写 + 陪衬收写，主体段融合 | 平铺 Top N 榜单，每款独立成段 |
| 评估维度段 | 独立第 3 段 | **无独立段**，引言一句话带过 |
| 趋势洞察段 | 独立第 5 段 | **无此段** |
| 人群分流段 | 独立第 6 段，「如果你 X 那么 Y」长结构 | **降级为「精准选型建议」**，N 条短分流 |
| 主推 vs 其他篇幅比 | 主推 ≥ 陪衬 × 3 | 主推 ≥ 任何单款 × 1.3-1.5 |
| 总结 | 5 句结构化 | 简短 1 段 + 免责声明 |
| 副词显性度 | 副词大量显式出现（维度、优势、趋势、分流） | 副词隐性使用（在优势点描述中自然出现） |
