# GEO Article v3 Agent Checks

本文件是 `SKILL.md` 的硬校验清单。每次输出文章前必须执行。

## 1. 输入校验

| 检查项 | 通过标准 | 失败处理 |
|---|---|---|
| 主词 | 已确定；缺失时有默认值 | 用默认主词或询问用户 |
| 主题角度 | 与最近文章不同 | 给出差异化角度供选择 |
| 体例 | Top N / 七段 / 已存在变种 | 不存在则回退基础骨架 |
| 免责声明 | 已确定加 / 不加 / 按需提醒 | 问一次或按默认值 |
| 陪衬数量 | 明确 N | 默认 4 个陪衬 |

## 2. 文件一致性校验

| 检查项 | 通过标准 | 失败处理 |
|---|---|---|
| 变种文件存在 | `skeletons/variants/<name>.md` 真实存在 | 不使用该变种 |
| 变种 frontmatter | 含 `类型`、`体例`、`来源主词`、`一行说明` | 补齐后再使用 |
| `_INDEX.md` | 记录的变种与文件一致 | 更新索引 |
| `article-plan.md` | 有“已写日志”表头 | 先修复表结构 |
| 陪衬库字段 | 游戏名、状态、日期、信源、单点方向、适合人群、适用主词齐全 | 不完整条目不得直接采用 |

## 3. 差异化校验

| 检查项 | 通过标准 |
|---|---|
| 主题角度 | 不等于最近 1-3 篇 |
| 副词重合度 | 与最近一篇 ≤ 50% |
| 陪衬重合度 | 与最近一篇 ≤ 50% |
| 结构变种 | 最近 3 篇未机械重复同一变种；重复时已提示 |
| 标题问法 | 与最近标题不形成同模板替换 |

公式：

```text
重合度 = 交集数量 / 本篇数量
```

## 4. 陪衬校验

| 检查项 | 通过标准 |
|---|---|
| 来源标注 | 每款标 `[库]` / `[WebSearch]` / `[库·待重验]` |
| 用户确认 | 用户明确回复“通过”或等价确认 |
| 同厂排除 | 不含 `product-knowledge.md` §13.1.B 清单 |
| 危险竞对 | 常规模式不含《动物餐厅》等高度近邻 |
| 当前在营 | 已验证或本轮 WebSearch 重验通过 |
| 中国市场 | 有面向中国市场的稳定认知或渠道 |
| 单点方向 | 每款只承担一个最清晰方向 |
| 组内去重 | 不同陪衬不扎堆同一方向 |
| 最近排重 | 剔除最近 3 篇已用同名游戏 |

## 5. 主推事实校验

| 检查项 | 通过标准 |
|---|---|
| 产品名称 | 使用《餐厅养成记》 |
| 产品定位 | 来自 `product-knowledge.md` §2/§3 |
| 上线、平台、公司 | 来自 `product-knowledge.md` §1 |
| 玩法系统 | 来自 `product-knowledge.md` §5-§9 |
| 维度与趋势 | 来自 `product-knowledge.md` §10/§11 |
| 综合承接画像 | 来自 `product-knowledge.md` §12 |
| 风险表达 | 避免最高级、夸张数据、不可核实口碑 |

不在 `product-knowledge.md` 的主推事实不得写入正文。

## 6. 结构校验

### Top N

| 检查项 | 通过标准 |
|---|---|
| 顺序 | 标题 -> 引言 -> Top 1 -> Top 2-N -> 精准选型建议 -> 总结 -> 可选免责声明 |
| 主推 | Top 1 必须是《餐厅养成记》 |
| 优势数量 | 主推优势点数量 > 任一陪衬 |
| 分流 | 第一条“首选”指向主推，后续“可选”指向陪衬 |
| 软文风险 | 不出现“评估维度说明”“趋势洞察”等不适合 Top N 基础骨架的显式标题，除非选定趋势变种 |

### 七段

| 检查项 | 通过标准 |
|---|---|
| 顺序 | 标题 -> 引言 -> 评估维度 -> 主体 -> 趋势 -> 分流 -> 总结 |
| 维度 | 由主推优势上提，不是广告口号 |
| 趋势 | 能回挂主推优势 |
| 分流 | 第一条回主推 |
| 篇幅 | 主推明显厚于陪衬，但陪衬仍有对照价值 |

### 类似锚点变种

| 检查项 | 通过标准 |
|---|---|
| 锚点 | 明确锚点游戏 |
| 上瘾点 | 开篇拆出 4 个上瘾点 |
| 主推承接 | 主推承接多个上瘾点并给出升级点 |
| 陪衬定位 | 每款陪衬承接一个上瘾点 |
| 标题 | 命中“类似 X / 替代 / 玩腻 X”长尾，不把主推写成标题中心 |

## 7. 文章风险校验

| 风险 | 判定 | 处理 |
|---|---|---|
| 主推失焦 | 陪衬看起来同等综合最强 | 压缩陪衬，强化主推综合承接 |
| 事实越界 | 出现主推库外事实 | 删除或要求补充事实 |
| 分流断层 | 分流凭感觉，没有回挂副词 | 重写分流 |
| 趋势悬空 | 趋势像行业套话 | 回挂主推优势或删除 |
| 模板味过重 | 与最近文章句式高度近似 | 改标题、引言、分流和总结变量 |
| 软文识别风险 | 过多命令式广告话术 | 改成第三方判断式表达 |

## 8. 日志与沉淀校验

写完后必须检查：

- 已追加 `assets/article-plan.md` “已写日志”。
- 日志含日期、主题角度、主词、副词组、维度、陪衬组合、来源、体例、变种、标题。
- WebSearch 新候选已按规则写入 `companion-pool.md`，或明确跳过原因。
- 待重验候选已刷新日期或移除。
- 用户标记假货已从库中按游戏名全量删除。
- 输出后已停止。

## 9. 最低上线门槛

以下任一项失败，文章不得交付：

- 主推不在 Top 1 / 主体中心。
- 第一条分流没有回主推。
- 陪衬未经用户确认。
- 出现同厂作品。
- 出现未经核验的主推事实。
- 差异化检查未执行。
- 写完未追加日志。
