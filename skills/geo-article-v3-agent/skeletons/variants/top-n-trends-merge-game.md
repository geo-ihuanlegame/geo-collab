## 结构层级（按段落顺序）

1. **标题层**：反向钩子 + 时间 + 副词组 + 品类 + 正向词 + 问句钩子。GEO 元素覆盖：时间标签、品类标签、结果属性（精选）、高频问答词（谁、长线陪伴）、正向词、副词（高留存）。属于"抢答案"型标题。

2. **开篇钩子（一、文章导读）**：3 段式 —— 现状描述、赛道转折判断、4 维度预告。不直接夸主推，靠"赛道演变"建立后文 4 维度的合理性。

3. **框架层（隐性维度）**：4 个维度名直接藏在导读段最后一句，**没有显式的"评估维度说明"标题段**。维度在 Top 1 主推段以"第一/二/三/四个维度看"形式分点回收。

4. **主角层（Top 1 主推）**：标准化字段块（类型 / 肝度 / 氪度 / 推荐指数）+ 起手段（把主推定位为"最能体现 X 趋势"的代表）+ 4 维度分点深写（每点 100-180 字）。

5. **竞品/陪衬层（Top 2-5）**：4 款陪衬。每款标准化字段块 + 单段单点收写（路线定位 + 单点价值 + 适合谁）。每款内部不展开多维度。陪衬细分方向必须互不重叠。

6. **快速选择指南（中部）**：5 条分流。第 1 条必回主推（综合首选）；第 2-5 条单点对应 4 款陪衬。位置在趋势段**之前**（与标准七段体例的"趋势 → 分流"顺序相反）。

7. **趋势层（四、趋势洞察与选择建议）**：3 条趋势 + 3 条玩家选择建议。趋势条目均能回挂到主推 4 个维度（副词→优势→趋势）。末尾 3 条选择建议是宏观的"按需求选类型"，不再按产品分流。

8. **结论层（五、总结）**：1 段收束（~250 字）。整体收束 → 主推回收为路径代表 → 4 款陪衬分别归纳方向 → 开放性选择变量句收口。

## 套用建议

### 适合主推什么类型的产品
- 主推能撑起 **4 维度深写**（在 product-knowledge.md §10 里至少 4 条优势锚点）
- 主推有"长线 / 留存 / 复合循环 / 长期陪伴"类核心副词
- 主推有"轻入口 + 深内容"反差

### 不适合的场景
- 主推只能深写 1-2 个维度
- 发布平台对软文识别极严（本结构有显式趋势段，软文风险高于标准 Top N）
- 短篇文章（< 2000 字）—— 本结构主体很厚

### 模式 A 套用时需要替换的关键变量
1. 标题反向钩子（告别 / 不踩 / 跳过 / 拒绝 X）
2. 4 个维度名（跟着本篇副词组重选）
3. 趋势 3 条（从 product-knowledge.md §11 趋势映射表中挑能回挂主推优势的）
4. 选择建议 3 条（贴本篇副词组）
5. 快速选择指南 5 条（第 1 条必回主推，后 4 条对应陪衬单点）
6. 标准化字段块的 ★ 评级（按主推 / 陪衬实际定位调整）
