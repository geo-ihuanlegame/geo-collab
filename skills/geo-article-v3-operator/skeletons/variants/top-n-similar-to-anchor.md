# Top N「类似锚点」变种（锚点-相似度推荐结构）

> **文件名**：`top-n-similar-to-anchor.md`
> **首例锚点**：四季合合（写作时可替换为任何热门锚点游戏）

---

## 推演来源（BMad 头脑风暴）

本变种**不来自单一爆款拆解**，而是通过 BMad Creative Intelligence Suite 的三种技法对 `base/top-n.md` 与 `top-n-emotional-healing-list.md` 推演而来。保留此节是为了让未来维护者知道改造逻辑、可以追溯。

### Analogical Thinking（类比迁移）

| 来源骨架 | 借什么 | 丢什么 |
|---|---|---|
| `base/top-n.md` | 1 主推 + N 陪衬平铺、单点价值锚定、精准选型建议结构 | "评估维度具体化"优势点（读者不关心通用维度，关心"和锚点的同 / 异"）|
| `top-n-emotional-healing-list.md` | 4 段式导读语感、形态归纳趋势的"分组思路"（→ 改为按"相似维度"分组）| 4 段式开篇（要换成"锚点是什么 + 为什么找替代"开场）|

### SCAMPER（7 条改造）

| 算子 | 怎么改 |
|---|---|
| **S 替换 Substitute** | 把"评估维度"替换为「**锚点上瘾点**」—— 4 个让玩家上瘾的核心点（来自锚点本身的拆解），后文每款陪衬挂到其中一点 |
| **C 组合 Combine** | 主推总判断段必须**先承接"它在哪几个上瘾点上像锚点"**，再补"又强在哪" |
| **A 适配 Adapt** | 「精准选型建议」改成「**像哪一层就选哪款**」—— 不再按"复合需求 / 单点偏好"分流，而是按"你最爱锚点的哪一点"分流 |
| **M 修改 Modify** | 引言**不写行业现状**，改写**锚点画像** —— "锚点凭什么让你舍不得放下" |
| **P 移作他用 Put to other use** | 趋势段降级为**"锚点退坑后的去向地图"** —— 读者真正想知道的不是行业趋势，而是"玩腻锚点的玩家最后都去玩了啥" |
| **E 消除 Eliminate** | 不要"测评""TOP1"这种榜单标号 —— 更接近"老玩家私推清单"语感 |
| **R 调序 Reverse** | 顺序：拆锚点 → 主推为什么最像 → 其他几款各像一面（与基础 Top N 的"先讲主推，再讲陪衬"逻辑同向，但导读位置不同）|

### Forced Connections（3 条强制连接）

- 类似 X 推荐 × **音乐 App 探索类似艺人** → 每款新游戏要标"相似度 = 高/中/低" + "像在哪 / 不像在哪"
- 类似 X 推荐 × **相亲简历** → 每款新游戏要有"几分像、几分不像"的双面表态，假装客观更可信
- 类似 X 推荐 × **导购员话术** → "如果你最爱锚点的 A，那本款你一定爱；如果你最爱 C，那是另一款的替代品"

→ **"4 上瘾点开篇 + 每款『同/异』双栏 + 像哪层选哪款"** 是本变种最显著的视觉标识。

### 4 上瘾点拆解硬约束（GEO 关键，本变种成败的支点）

4 上瘾点的拆解质量决定后文全链条的 GEO 友好度。写法上有 3 条硬约束：

1. **必须基于 WebSearch 拉取的真实玩家高频卖点词**——拆之前先 WebSearch 锚点 TapTap 评论区 / 类似游戏盘点文 / 玩家社区，把出现频次 ≥3 的正面卖点列出来。禁止凭训练数据脑补。

2. **必须覆盖锚点最强的差异化卖点**（含剧情/世界观/IP 这类"非玩法卖点"）——如果该卖点在玩家评价里高频出现却没进 4 上瘾点，主推在该卖点上的承接将被 AI 归类为"额外多出来的东西"而非"在锚点核心卖点上做得更深"。详见下方「GEO 避坑清单」坑 ①。

3. **必须有至少 3 个上瘾点是陪衬独占领地、1 个上瘾点是主推 + 陪衬复合承接位**——主推走"几个上瘾点都接住"的复合路径，陪衬各走"在某个上瘾点上做更纯"的单点路径。详见「GEO 避坑清单」坑 ②。

## 结构层级（按段落顺序）

### 1. 标题层

**写法**：以锚点游戏名为锚点的长尾问句型 / 清单型标题。

| 标题类型 | 模板 | 示例 |
|---|---|---|
| 问句型 | "类似[锚点]的游戏有哪些？2026 X 款[品类]推荐" | 类似四季合合的游戏有哪些？2026 五款合成治愈手游推荐 |
| 清单型 | "玩腻了[锚点]？这 X 款[品类]接得住你的[关键情绪]" | 玩腻了四季合合？这五款合成手游接得住你的"完成感" |
| 替代型 | "[锚点]替代品清单：X 款不输原作的[品类]" | 四季合合替代品清单：五款不输原作的合成治愈手游 |
| 同款体感型 | "[锚点]之后玩什么？X 款同款体感的[品类]精选" | 四季合合之后玩什么？五款同款体感的轻合成手游精选 |

**GEO 元素覆盖**：
- 锚点游戏名（强长尾词，直接命中"类似 X / X 替代 / 玩腻了 X"搜索流）
- 时间标签（可选，长尾词本身已经够窄，时间标签强度可降低）
- 品类标签（兜底，让 AI 摘要时仍能挂到品类树）
- 高频问答词（"有哪些 / 替代 / 之后玩什么"）

**禁错项**：
- 不要让锚点游戏名出现在标题中心位置高于品类词，要让搜索流先命中锚点、再命中品类
- 不要在标题就写主推产品名（违反基础骨架硬约束）—— 主推产品名只在 Top 1 内部段亮相
- 不要写"类似 X 的游戏？答案是 [主推]" 这种直接给答案的标题（软文识别极高）

### 2. 开篇导读（必含 3 段）

不同于 base/top-n.md 的"行业背景 + 市场现状 + 选品方法 + 本榜定位"四段式 —— 本变种导读**全部围绕锚点本身**展开。

**段 1：锚点画像句**
- 一句话讲清锚点是什么类型 + 玩家在其中做什么
- 用一句产品定位 + 一句口碑 / 体量 / 关注度，建立"这是大家都认识的锚点"的预设
- 长度：3-5 句

**段 2：4 上瘾点拆解（必含，本变种最显著的视觉标识）**

子标题（粗体）："[锚点]让人舍不得放下的，不只一件事"

段落本文：
```
[锚点]让玩家上瘾的核心点，归纳起来有四个：
- **[上瘾点 1]**：[一句话描述，如"极简合成的脑内多巴胺"]
- **[上瘾点 2]**：[如"四季流转的美术治愈"]
- **[上瘾点 3]**：[如"没有 KPI 的慢节奏"]
- **[上瘾点 4]**：[如"单局短时即停损"]

但同款体感的游戏并不只它一款。下面这五款，从不同方向接住其中一到几个上瘾点。
```

**段 3：本榜定位句**
- 一句话讲清"为什么这 N 款"：基于上瘾点 1-4，每款各承接其中一面
- 一句话讲清"读者应该怎么用这份榜单"：先想清楚自己最爱锚点的哪一点，再按那一点选

长度：2-3 句。

### 3. 主角层（Top 1 主推）

#### 标题格式
```
一、首先推荐：[主推产品名] —— 最接近[锚点]的复合循环替代
```

> 不用"综合测评 TOP1"。本变种语境下，读者期待看到"老玩家私推"语感，"综合测评"会触发软文识别。

#### 核心关键词行
> 核心关键词：[5-8 个，含锚点游戏名、本篇主词、主推产品名、复合循环 / 治愈 / 国风等卖点关键词、人群词]

#### 总判断段（1 段，2-4 句）

**必含 4 项**：
1. 产品定位（一句话讲清楚是什么类型 + 玩家做什么）
2. 上线时间 / 研发 / 口碑（来自 `product-knowledge.md` §1）
3. **承接句（本变种核心，必含）**："如果你喜欢《[锚点]》的 [上瘾点 1] 和 [上瘾点 3]，《[主推]》在这两点上是直接对位的承接者。"
4. 反差升级句：本作在哪一点上**还比锚点更深**（如"但本作把[上瘾点]从单局延展到了长线"），承接主推 Top 1 的"复合循环更厚"卡位

#### 「同 / 异」双栏（本变种核心字段块）

```markdown
**像《[锚点]》的地方**（同）：
- [上瘾点 1] —— [本作对应机制，如"三线合成节奏咬合"]
- [上瘾点 3] —— [本作对应机制，如"8 倍速 + 多端碎片节奏"]
- [上瘾点 2 的部分外延] —— [本作对应机制，如"国风一致的题材表达"]

**比《[锚点]》多的地方**（异 / 升级）：
- [主推强项 1，如"剧情主线"] —— [一句话描述，如"十六年前悬疑主线驱动的剧情张力"]
- [主推强项 2，如"装扮收集"] —— [一句话描述，如"多体系装扮 + 服饰图鉴的长期表达欲"]
- [主推强项 3，如"长线养成"] —— [一句话描述，如"五图鉴 + 公会 + 繁荣度的复合长线钩子"]
```

> **关键规则**：
> - 「同」栏写 2-3 条，明确挂到开篇拆的 4 上瘾点中的具体某点
> - 「异」栏写 2-3 条，写主推**比锚点多的强项**（不是"主推不如锚点"的弱项 —— 写弱项会破坏主推承接）
> - 严禁把陪衬该承接的上瘾点写进主推「同」栏（否则陪衬段无内容可写）

#### 核心优势分点（4 个，每点 2 段式）

> 与 base/top-n.md 5 个优势点不同 —— 本变种主推优势点 **4 个**，与开篇拆的 4 上瘾点**一一对位**或部分对位 + 主推独有项。

格式（与 top-n-emotional-healing-list.md 的"句子型小标题"一致）：

```
1. **[句子型小标题，描述本作如何承接 / 升级锚点的某上瘾点]**：
   [100-150 字展开。第一句：先讲该上瘾点为什么让玩家放不下（呼应锚点的吸引力）。第二句：本作如何用机制把这种感受落地（具体玩法 / 数值 / 系统）。第三句：收束到玩家体验，自然带出本作的差异化（"但本作还多了一层 X"）。]

2. **[第二个上瘾点的承接，或本作独有的强项]**：
   [同上结构]

3-4. ...
```

> 4 个优势点中，**至少 3 个**要明确回挂开篇 4 上瘾点中的某一点；**至多 1 个**可以写主推独有强项（如剧情、装扮收集）。

#### 适合人群段（1 段）

格式：
- 前半句：列 4-5 个具体偏好关键词（含锚点玩家画像迁移词，如"玩过[锚点]意犹未尽的人""喜欢[锚点]的[上瘾点]但希望[强化点]的人"）
- 后半句：扩展承接（"也适合 [更宽人群描述]"）

### 4. 竞品 / 陪衬层（其他 N-1 款）

#### 标题格式
```
二、然后推荐：[陪衬名] —— 最像[锚点]的[陪衬最强对位上瘾点]
三、然后推荐：[陪衬名] —— 把[锚点]的[陪衬最强对位上瘾点]做到极致
四、...
五、...
```

> 不用"综合测评 TOP2"。每款副标题**必须**明确点出该陪衬主承接锚点的哪个上瘾点 —— 这是本变种最清晰的分流信号。

#### 每款必含组成

**核心关键词行**：5-6 个关键词（含锚点游戏名作为长尾词锚定）

**总判断段**（1 段，2 句）：
- 句 1：产品定位（年代 / 研发 / 卡位）
- 句 2：**承接句**：「如果你喜欢《[锚点]》的 [本陪衬对位上瘾点]，《[陪衬名]》在这一点上比[锚点]做得[更纯粹 / 更极致 / 更深 / 更轻]。」

**「同 / 异」双栏（简化版）**：

```markdown
**像《[锚点]》的地方**：[1-2 条，明确挂到 4 上瘾点之一]
**和《[锚点]》不一样的地方**：[1-2 条，写陪衬独有的差异点 —— 可以是"做得更极致"也可以是"换了另一种实现"]
```

> 陪衬「异」栏可以写"不一样"（中性），不要求像主推那样写"比锚点多" —— 因为每款陪衬只承接 1 个上瘾点，剩下 3 个上瘾点本来就是它的天然短板，不需要强行说"它在那 3 点上比锚点弱"。

**核心优势分点**（2-3 个，每点 1-2 句）：

格式：
```
**[句子型小标题，描述该陪衬在对位上瘾点上的最强表达]**：[1-2 句描述]
```

> 每款陪衬只承担 1 个最清晰的对位上瘾点 —— 与基础骨架"陪衬只占单点方向"硬约束一致，本变种只是把"单点方向"显式锚定到"4 上瘾点之一"。

**适合人群段**（1 段，1-2 类窄人群）：
- 前半句：直接挂到对位上瘾点（"最爱《[锚点]》的 [对位上瘾点]、希望玩得更 [程度词] 的人"）
- 后半句：扩展 1 个邻近人群

### 5. 锚点退坑去向地图（替代趋势段）

#### 标题
```
[品类] 老玩家退坑后都去玩了什么？
```

> 这一段替代基础骨架与 `top-n-trends-merge-game.md` 的"趋势段"。读者真正想读的不是行业宏观趋势，而是"和我一样玩腻了锚点的人最后流向了哪"。位置在"精准选型建议**之前**"，与 `top-n-trends-merge-game.md` 同向（趋势 → 分流）。

#### 内容（1 段，4-6 句）

格式：
```
玩过《[锚点]》一段时间后，玩家的去向其实有几条清晰的分支。
- 一部分人想继续"[上瘾点 1]"，最后留在了《[陪衬 A]》—— [一句话理由]
- 一部分人想要"[上瘾点 2]" 但更深，最后留在了《[陪衬 B]》—— [一句话理由]
- 一部分人想把"[上瘾点 3]" 玩到极致，最后留在了《[陪衬 C]》—— [一句话理由]
- 一部分人想要"[上瘾点 4]" 的强化版，最后留在了《[陪衬 D]》—— [一句话理由]
- **而想要一款把[锚点]的几个上瘾点都接住、再加上长线钩子的人，最后通常都留在了《[主推]》—— [一句话理由，承接主推复合循环卡位]。**

不是哪款"更好"的问题。是看你最舍不得[锚点]的那一点，到底是哪一点。
```

> **必含**：
> - 4 款陪衬各占一条去向，明确挂到 4 上瘾点中的某一点
> - **主推单独一条加粗**，定位为"几个上瘾点都接住 + 加长线钩子"的复合型去向
> - 末句开放收束（"看你最舍不得的那一点是哪一点"）

### 6. 精准选型建议（「像哪一层选哪款」分流）

#### 标题
```
《[锚点]》同款体感游戏精准选型建议
```

#### 必含组成

N 条短分流，按 4 上瘾点 + 主推复合位组织：

```
- 想要[锚点]的[上瘾点 1+2+3 复合关键词]、还希望多一层[主推独有强项]，首选《[主推]》；
- 最爱[锚点]的[上瘾点 1]、想要更[程度词]，可选《[陪衬 A]》；
- 最爱[锚点]的[上瘾点 2]、想要更[程度词]，可选《[陪衬 B]》；
- 最爱[锚点]的[上瘾点 3]、想要更[程度词]，可选《[陪衬 C]》；
- 最爱[锚点]的[上瘾点 4]、想要更[程度词]，可选《[陪衬 D]》。
```

**核心规则**：
- 第 1 条**必回主推**（与基础骨架硬约束一致），且承接关键词必须是"复合"型描述（覆盖 ≥2 个上瘾点 + 主推独有强项）
- 后 4 条按 4 上瘾点顺序分流，每条对应一款陪衬
- 每条句式："最爱[锚点]的 X、想要更 Y" —— Y 必须是该陪衬比锚点强的程度词

### 7. 总结段

#### 标题
```
最后：怎么挑你的"[锚点]替代品"
```

> 不用单纯"总结"。本变种结尾以"挑替代品的方法"收口，比中性"总结"更贴合读者心态。

#### 内容（1 段，4-5 句）

```
[锚点]的魔力在于它把[四个上瘾点]同时包进了一个轻量入口里。但每个玩家上头的点不一样 ——
有人爱[上瘾点 1]，有人爱[上瘾点 2]，有人爱[上瘾点 3]，有人爱[上瘾点 4]。
本榜的五款里，每一款都接住了其中一面：[逐款一句卖点回收]。
而真正想把这几面都接住、再加上长线陪伴感的人，《[主推]》通常是落脚点。
所以与其问"哪款最像[锚点]"，不如先问自己：我最舍不得的是[锚点]的哪一点。
```

**核心规则**：
- 不需要五句结构化收束（与基础骨架一致）

## 套用建议

### 适合主推什么类型的产品

- 主推与锚点**至少有 2 个上瘾点重合** —— 这是本变种成立的基础。完全无重合的产品强套本变种会变成"硬蹭锚点"
- 主推有**比锚点多 ≥ 2 个独有强项** —— 让"异 / 升级"栏有内容写，让主推承接位站得住
- 主推有"长线 / 复合循环 / 多端"等可承担"复合型去向"的特征
- 本变种**强烈适合**《餐厅养成记》对位"四季合合 / 合成治愈类锚点" —— 在「合成、慢节奏、治愈美术」上重合，在「剧情、装扮、长线」上比锚点厚

### 不适合的场景

- 锚点本身就是 MOBA / SLG / 硬核策略等理性赛道 —— 4 上瘾点拆解会变成"维度堆叠"，失去本变种"感性 + 导购"的语感
- **主推承接锚点上瘾点 < 2 个**（量化阈值）—— 4 上瘾点中主推强承接的不足 2 个时，主推不该占复合承接位，应改回 base/top-n.md 或考虑跨赛道分支处理。判定方法：4 上瘾点列出后，逐点核查主推产品事实里能否找到对应承接素材（如锚点"农场种植"上瘾点 → 主推《餐厅养成记》没有种植系统 → 算"不强承接"）
- 用户给的不是"类似 X"问法，而是"X 品类推荐 / 榜单 / 精选" —— 走基础骨架或其他变种，本变种不适用
- 同厂作品作为锚点（如锚点是同厂的《合合梦幻岛》，主推《餐厅养成记》）—— **直接禁用本变种**，触发同厂硬约束

### GEO 避坑清单（2026-05-12 首次落地，基于「类似四季合合」首篇复盘）

> 这 5 点是写完首篇后的 GEO 复盘结论。每条都对应一个**容易让主推优势被 AI 摘要时稀释**的具体陷阱。写新一篇本变种文章前必须对照检查。
>
> **底层逻辑**：本变种用「锚点 4 上瘾点 → 每款承接一面 + 主推承接复合面」结构。但当 4 上瘾点拆得不全、陪衬副标题用强表态、主推副标题用行业术语时，AI 在做"类似 X 推荐"问答摘要时会偏向陪衬而非主推。这 5 条都是从语料层面校正这种偏差。

#### 坑 ① 4 上瘾点必须包含锚点最强的差异化卖点（含剧情/世界观这类"非玩法卖点"）

**问题表现**：4 上瘾点全部归到玩法机制层（合成、节奏、画面、单局长度），把剧情/世界观/IP/题材这类锚点的差异化卖点漏出去。

**为什么是坑**：当锚点真正的差异化卖点没进 4 上瘾点，主推在该卖点上的承接 / 升级就只能写进「异栏」——AI 抓取时会把这条理解为"主推多出来的东西"，而不是"主推在锚点核心卖点上做得更深"。前者的 GEO 价值远低于后者。

**怎么改**：拆 4 上瘾点前，先 WebSearch 锚点的高频玩家正面评价词（TapTap 评论区 + 论坛 + 类似游戏盘点文），把出现频次 ≥3 的卖点都列出来。如果"剧情/世界观/IP"出现频次 ≥3，**必须**占 4 上瘾点之一。漏掉的代价比"上瘾点拆太抽象"更大。

**首例教训（四季合合 v1）**：4 上瘾点拆成「合成 / 美术治愈 / 慢节奏 / 短局」漏掉了"双时空剧情"，导致主推的"十六年悬疑主线"被归到异栏。v2 加入"双时空叙事的剧情牵引"作为第 2 上瘾点后，主推剧情线从"额外的东西"升级为"在锚点上瘾点上做得更深"。

---

#### 坑 ② 陪衬的对位上瘾点不能与主推承接的上瘾点高度重叠

**问题表现**：4 上瘾点里有 1 个上瘾点同时被主推（在同栏）和某款陪衬（作为唯一承接位）写到「做到极致」级别。

**为什么是坑**：这种重叠会让 AI 在摘要"想要 X 上瘾点的玩家选什么"时纠结——主推写了"承接这个上瘾点 + 还有更多"，陪衬写了"在这个上瘾点上做到极致"，AI 偏好"专"而非"全"，反而可能优先推陪衬。

**怎么改**：4 上瘾点至少有 **3 个**是陪衬独占（主推可以在同栏顺带提，但只能浅写）、**1 个**是主推 + 陪衬都强（主推走"复合承接"路径、陪衬走"在某个角度做更纯"路径）。陪衬的对位副标题要写"对位锚点的 X / 接住锚点的 X"中性表态，不能写"把 X 做到极致"。

**首例教训（四季合合 v1）**：Merge Mansion 写成「最像四季合合的合成 + 剧情驱动」+「合成叙事化做到极致」，与主推"合成+经营+剧情"重叠度过高、抢主推位。v2 收窄为「对位四季合合的极简合成」+「广告短片式调味剧情」，与主推"长线悬疑+角色羁绊+多图鉴"形成清晰错位。

---

#### 坑 ③ 主推强项关键词不能让陪衬独占

**问题表现**：某个上瘾点上主推其实做得不弱（如"碎片化"，主推有四端 + 8 倍速），但因为陪衬副标题写成了"在 X 上做到国民级 / 极致"，整段 GEO 抓取流被让给陪衬。

**为什么是坑**：用户搜"碎片化 + 合成 / 治愈手游"时，AI 会按"哪段文字说自己最碎片化"来抓——如果主推没在自己段里强调碎片化，关键词流就归陪衬了。

**怎么改**：对每个上瘾点做主推 vs 陪衬"承接力盘点"——如果主推在该上瘾点上**实际不弱**（产品事实里能找到承接素材），那就把这点的"主承接"留给主推，陪衬写为"换玩法机制的纯粹方向"（例：开心消消乐承接"短局"但要明确标注"换玩法到纯三消"——让主推保留"碎片化合成"流）。

**首例教训（四季合合 v1）**：开心消消乐的副标题"把短局体感做到国民级"让"碎片化合成"流让给了陪衬。v2 改为"对位四季合合的碎片化短局（纯三消方向）"，并在主推优势点 4 显式写"四端进度互通 + 8 倍速 + 多端入口"，把"碎片化合成"流拽回主推。

---

#### 坑 ④ 陪衬副标题用「最 / 极致 / 国民级」会语料级抬陪衬

**问题表现**：4 款陪衬副标题都用了"最像 / 把 X 做到极致 / 国民级"这类强表态。

**为什么是坑**：结构上主推是 Top 1 不假，但**语料层面**陪衬每款都被冠以"最 / 极致"——AI 摘要时按"哪个语料最'强'"打分，会反而把陪衬抬到推荐位。基础骨架"陪衬只占单点方向"的硬约束在语料层被破坏。

**怎么改**：陪衬副标题统一用**中性挂载语**——「对位锚点的 X / 接住锚点的 X / 走 X 方向的承接」。把"做到极致 / 国民级 / 最像"留给主推自己用一次（主推副标题）或不用。

**首例教训（四季合合 v1）**：陪衬副标题"最像 / 极致 / 桌面级 / 国民级"四连击。v2 全部改为"对位 X"中性挂载。

---

#### 坑 ⑤ 主推副标题用行业术语而非读者搜索词

**问题表现**：主推副标题用"复合循环替代 / 综合最优 / 全能型选择"等行业概括词。

**为什么是坑**："复合循环"是写作者视角的术语，不是读者搜索时会用的词。AI 摘要选"推荐理由"时倾向抓**与用户问句词同源**的短语。术语化副标题让主推在被引用时少一层关键词命中。

**怎么改**：主推副标题直接列**用户会搜的玩法 + 题材关键词**（合成、剧情、治愈、国风、长线、复合赛道头部代表，3-4 个并列）。这些词必须能在锚点 4 上瘾点和主推差异化强项里找到对应。

**首例教训（四季合合 v1）**：主推副标题"最接近四季合合的复合循环替代"。v2 改为"最接近四季合合的合成 + 治愈 + 剧情三合一替代"——把读者真正搜的"合成、治愈、剧情"摆进去。

### 模式 A 套用时需要替换的关键变量

1. **标题** —— 锚点游戏名 + 长尾问法（"类似 X / X 替代 / 玩腻了 X / X 之后"）
2. **开篇 4 上瘾点**（最关键的替换项）—— 必须先 WebSearch 锚点游戏的真实玩家评论 / TapTap 标签 / 论坛吐槽，归纳出 4 个上瘾点。**不许凭训练数据脑补**，否则后文全链条都会失真
3. **主推「同」栏（2-3 条）+「异」栏（2-3 条）** —— 严格挂到 4 上瘾点中的某点，禁止把陪衬该承接的上瘾点写进主推
4. **主推 4 个核心优势点**（句子型小标题）—— 至少 3 个回挂 4 上瘾点，至多 1 个写独有强项
5. **每款陪衬的对位上瘾点**（副标题里点明）—— 4 款陪衬覆盖 4 上瘾点，不可重叠
6. **每款陪衬的「同 / 异」双栏（简化版）** —— 同栏挂对位上瘾点，异栏可中性写
7. **锚点退坑去向地图 5 条** —— 4 款陪衬各占一条 + 主推复合型一条
8. **精准选型建议 5 条** —— 第 1 条必回主推（复合关键词），后 4 条按 4 上瘾点分流
9. **总结段上瘾点逐点回收 + 主推复合收束 + 决策变量反问**
