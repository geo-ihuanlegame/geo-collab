# 文章规划与已写日志（合一）

> 本文件是模板。第一次使用时复制为 `article-plan.md`（同目录），之后**每次模式 A 写完一篇都追加一行**到「已写日志」节。
>
> 用途：
> 1. **可选规划**：用户说「先规划一下」时填「待写区」
> 2. **持续日志**：每篇写完追加到「已写日志」，供下篇写之前做差异化检查
>
> **差异化检查规则**（详见 [`../references/writing-guide.md`](../references/writing-guide.md) §11）：
> 与最近 1-3 篇相比 ——
> - 主题角度**不可相同**
> - 高权重副词组**重合度 ≤ 50%**
> - 陪衬组合**重合度 ≤ 50%**
> - 稳定项保持，可变项必须变化（**跨体例**进行）

## 当前默认主词、陪衬数量、体例

> **默认主词**：模拟经营 —— 用户每次可临时覆盖（如「这次主词改成『治愈手游』」）。表格里主词列默认填「模拟经营」，可逐行改。
>
> **默认陪衬数量**：1 主推 + 4 陪衬（共 5 款）—— 用户可在指令中临时调整（如「这次配 6 款」「短文配 3 款」）。
>
> **默认体例**：Top N 榜单 —— 用户可在指令中临时切换为「七段对比」（关键词：「七段结构」「七段对比」「深度对比」「主推深写」）。详见 [`../SKILL.md`](../SKILL.md) Output Structures 章节。
>
> **默认免责声明**：按需提醒 —— skill 在步骤 0.5.4 提醒一次，等用户回答。改成「加」让 skill 跳过提醒直接追加；改成「不加」让 skill 跳过提醒不追加。
>
> **陪衬游戏不预填**，由 skill 在写每篇前**库优先召回 → WebSearch 兜底**：先查 [`../references/companion-pool.md`](../references/companion-pool.md) 已审库，库内候选凑得够就直接用（热启动），不够再 WebSearch 补差额（详见 [`../references/companion-games.md`](../references/companion-games.md) §6）。规划时「拟用陪衬」列可留空或填希望的细分方向。

## 字段说明

| 字段 | 说明 |
|---|---|
| **主题角度** | 不同篇必须不同。常见角度：深度评测 / Top N 榜单 / 横向对比 / 玩法解析 / 趋势观察 / 人群推荐指南 / 入坑首选榜 / 治愈系推荐 / 国风专题 |
| **主词** | 主题锚点，决定「这篇讲什么」。默认「模拟经营」，可改为具体长尾搜索词 |
| **高权重副词组** | 决定文章主轴的判断方向词。从 [`../references/product-knowledge.md`](../references/product-knowledge.md) §10 选 3-5 个 |
| **选用维度** | 七段体例：评估维度说明段使用；Top N 体例：隐性指导优势点选择 |
| **拟用陪衬** | 规划时留空或填希望的细分方向；实际游戏名由 skill 在步骤 2.5 库优先召回 + WebSearch 兜底筛选 |
| **陪衬来源** | 「已写日志」专属字段。写法 `库×3 + WebSearch×1`（半热启动）/ `库×4`（热启动）/ `WebSearch×4`（冷启动）。便于回头审计哪几篇靠库、哪几篇是新血 |
| **体例** | 「Top N 榜单」（默认）或「七段对比」 |
| **同篇慎用核查** | 必须查 [`../references/companion-games.md`](../references/companion-games.md) §4 |
| **使用结构变种** | 本篇是套基础骨架还是 skeletons/variants/ 里的变种。值为 `基础骨架` 或具体变种文件名（如 `top-n-trends-merge-game.md`）。差异化检查时若最近 3 篇反复用同一变种，需提示用户考虑切换以避免段位手感同质化 |
| **状态** | 待写 → 已写 |

## 差异化判断快查

**重合度怎么算**：

```
副词重合度 = (本篇副词 ∩ 最近一篇副词) / 本篇副词
陪衬重合度 = (本篇陪衬 ∩ 最近一篇陪衬) / 本篇陪衬
```

**示例**：
- 最近一篇副词组：`[国风, 剧情, 女性向]`（3 个）
- 本篇副词组：`[国风, 治愈, 长线养成]`（3 个）
- 交集：`[国风]`（1 个）
- 重合度 = 1/3 ≈ 33% ✅ 通过（≤ 50%）

```
本篇副词组：[国风, 剧情, 治愈]
最近一篇：  [国风, 剧情, 女性向]
交集：     [国风, 剧情]
重合度 = 2/3 ≈ 67% ❌ 不通过，需调整
```

**触发不通过时 skill 的动作**：
1. 提示用户：「本篇与最近一篇 [副词/陪衬] 重合度 X%，超过 50% 阈值，需要调整」
2. 给出调整建议（替换哪些副词 / 哪些陪衬）
3. 等用户确认调整方案后再开写

## 滚动日志维护

- 写完一篇 → skill 自动追加一行到「已写日志」节
- 文件持续累积，无需清理（100 行内都不影响检索）
- 用户每次启动新一轮写作时，skill 默认对照**最近 3 行**做差异化检查
- 用户可手动修改 / 删除日志（比如某篇没真发，可删除对应行）
- **跨体例差异化**：差异化检查跨体例进行 —— 即使本篇用 Top N 榜单、上一篇用七段，主题角度 / 副词 / 陪衬组合的重合度依然按规则比较

---

## 与稳定项 / 可变项的关系

差异化检查**只针对可变项**。稳定项每篇都一样，不参与差异化判断。

| 稳定项（不变） | 可变项（必须变） |
|---|---|
| 当前体例下的段落顺序 | 主题角度 |
| 主推 = Top 1 / 主体中心 | 副词组 |
| 第一条分流（或精准选型建议第 1 条）回主推 | 维度命名 |
| 主推承担最多信息量 | 趋势名（七段体例） |
| 危险竞对默认排除 | 分流偏好句 |
| —  | 总结变量表达 |
| —  | 标题中的高频问答词 |
| —  | 体例本身（可在 Top N / 七段间切换） |
