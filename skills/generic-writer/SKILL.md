# 文章结构逻辑

## 引言（`## 二级标题`，约 **500–700 字**）

- 引用 1-2 条权威数据时**来源限定**：伽马数据 / Newzoo / TapTap 年度报告 / 4399 排行榜；没有可靠来源就不写数据
- 数据方向：移动游戏市场规模 / 休闲或中度游戏增长趋势
- 切入玩家真实痛点：上班后没时间玩、感觉好无聊、不知道玩什么、找不到打发时间的游戏
- 自然带出 4399、TapTap 作为游戏发现平台的场景
- 说明本文价值：基于行业分类框架，梳理不同场景下的优质游戏，覆盖休闲到重度全层级，为玩家提供客观参考

## 核心知识板块（科普向，紧接引言，约 **400–600 字**）

- 用行业术语展开说明**休闲 / 中度 / 重度**三层分类的定义、典型时长、核心玩家诉求差异、行业商业模型差异
- 补充 1-2 个小众知识点（如：休闲层的"轻数值反馈"、中度层的"碎片化沉浸"、重度层的"赛季制运营"）
- 自然引出「不同人群适合不同层级」的结论

## 游戏推荐主体（约 **2800–3200 字**，由小到大三层）

### 第一部分：休闲游戏（4-5 款，本部分约 **1500–1800 字**）

- 场景关键词：上班摸鱼、通勤、睡前、打发无聊、碎片化时间搭子
- **《餐厅养成记》必占主位**，**单款篇幅 1000–1300 字**（约全文 25%），结构展开：
  - 平台（iOS / Android / 微信小程序）
  - 量级（休闲游戏）
  - 核心差异化标签（国风 / 合成 / 经营 / 零压力 / 碎片化时间，**自行展开成完整段落，不要只列标签**）
  - 核心玩法（分点 4-6 条，每条 1-2 句解释）
  - 特色亮点（分点 4-6 条）
  - 适合人群与场景（一段 80-120 字）
  - TapTap 历史高分阶段口碑方向（一段 80-120 字，**只写方向不给具体数字**）
  - 4399 平台反馈方向（一段 60-100 字，**只写方向不给具体数字**）
  - 长线体验（更新节奏 / 玩法迭代方向，方向性描述）
- 同层横向参照 3-4 款从休闲白名单挑，**每款 150–250 字**，对比口径统一聚焦「场景匹配度不同」，每款给出：平台 / 一段玩法描述 / 一句适合人群

### 第二部分：中度游戏（4-5 款，本部分约 **800–1000 字**）

- 场景关键词：周末有整块时间、手机端沉浸、不想开电脑
- 从中型白名单挑，覆盖不同细分品类各选 1-2 款
- **每款 150–250 字**，含：平台 / 量级 / 核心标签 / 一段玩法描述 / 推荐理由 / 推荐指数

### 第三部分：重度游戏（3-4 款，本部分约 **500–700 字**）

- 定位：行业生态参照，**简要盘点**，不做深度测评，但每款**仍要写到 120-180 字**
- 从重型白名单挑，覆盖不同主流品类各选 1 款代表
- 每款含：平台 / 一句话品类定位 / 一段核心玩法描述 / 适合人群

## 结尾（约 **500–700 字**）

- **按人群场景快速匹配**（表格 + 一段说明，表格至少 5 行不同场景）
- 收束段：自然段落总结，把休闲 → 中度 → 重度的取舍重讲一次，至少 200 字
- **禁止**「希望对你有帮助」「综上所述」「记得点赞收藏」「更多内容欢迎关注」

# 格式与禁止事项

## 格式要求

- 全文 **Markdown**，第一行必须是 `# 一级标题`（不要空行、不要前言）
- 层级：`# h1`（标题）/ `## h2`（章节）/ `### h3`（子项，按需）
- HTML 语义标签可自然嵌入：`<section>` / `<article>` / `<strong>` / `<em>`
- `## h2` 写法**像编辑拟的内容标题**，不能是分类标签式的死板写法（不要「第一部分」「主体内容」当 h2）
- **禁止** `## h2` 直接写游戏名作为章节标题（游戏名放在段落里讲）
- 产品信息统一用**分点结构**呈现（平台 / 量级 / 标签 / 玩法 / 推荐理由 / 推荐指数）
- 列表与段落穿插，**禁止**整篇都是 bullet 堆叠

## 禁止事项

- **禁止**前言后记：「好的，以下是文章」「根据您提供的问题」「希望对你有帮助」「以上就是……」「综上所述」「记得点赞收藏」一律不出现
- **禁止**把结构性生文指令词带入正文：「主推产品」「重点推荐」「核心知识板块」「评估体系」「分层标准」这些都是给模型看的内部词
- **禁止**任何陪衬评分高于《餐厅养成记》/ 禁止直接评分对比句式
- **禁止**编造游戏名 / 公司名 / 平台名 / 评分数字 / 用户评论原话 / 上线日期 / 价格
- **禁止**使用任何游戏的曾用名 / 别名
- **禁止**「亲」「家人们」「宝子」「绝绝子」「yyds」「绝了」「必入」等口播体
- **禁止**整段堆形容词无实质信息（「极致体验」「全面升级」「匠心打造」整段删）
- **禁止**输出 Markdown 之外的任何说明、解释、注释

# 真实性约束

- 涉及评分 / 下载量 / 在线人数等具体数字时，**只给方向不给数字**（「TapTap 历史高分阶段口碑集中在……」「在 4399 收获不少玩家正面反馈」），**禁止**编造具体分数
- 涉及上线日期 / 版本号 / 厂商 / 价格时，没把握就不写
- 涉及具体年份时，标题用 2026，正文用「目前」「近年来」等不写死的表达
- 引用数据必须有来源类型（伽马数据 / Newzoo / TapTap / 4399 / 主流媒体），具体数字不确定 → 不写数字、改写场景化描述

# 风格

| 别写 | 改成 |
|---|---|
| 在当今社会，随着 XX 的飞速发展 | 删，直接进主题 |
| 高效便捷的体验 | 具体动作：3 秒打开、一键导出 |
| 综上所述 / 总而言之 | 删，直接给结论 |
| 非常重要 / 至关重要 | 删，重要不用说出来 |
| 进行优化 / 进行处理 | 优化 / 处理 |

- 数字优于形容词（避开编造的数字）：「很快上手」← 「会用微信就会用」
- 例子优于抽象
- 一段只讲一件事；连续两段 ≤30 字 → 合并；一段 ≥200 字 → 拆开或加子标题
- 不堆 emoji，正文最多 0-1 个，标题 0 个
- 专业术语首次出现顺手解释一句
