AI"大脑"不在 GEO 里跑，跑在 Claude Code 本机。GEO 只负责"能力"—— 生文、配图、打分、分发、推飞书。AI 自己规划节奏、自评、自纠，做完了在飞书群里报工。 跑得不好下次自己改、跑得好就稳定产出，运营人只管定目标和最终点 approve。

核心架构

左·运营人 → 中·AI 大脑 → 右·GEO 平台 · 三柱单向数据流 + 飞书 webhook 回报

图例： 现在能跑（实线） v3 愿景（虚线）v3 飞书回报（绿） 色系：蓝=流程 · 紫=素材 · 绿=数据 · 青=对外接口

一次"生文 Loop"跑了什么

6 步串起来 · 老板看个流程感 · 跑完结果直接进未审核库等人审

1
启动
运营人本机敲 /loop
→ 飞书弹"Loop 开始"
2
拉资料
AI 找今天可写的问题 + 可用模板
→ 从 GEO 问题池 / 提示词库读
3
写 + 配图
一篇一篇生 · 顺手配好图
→ 沉到 GEO 草稿库
4
自评
让另一个便宜 AI 给文章打分
→ 事实 / 可读 / 风格 / 合规 四维
5
决策
≥70 进未审核库 · 低分自动重写或弃
→ 人审兜底，不直接发
6
报工
飞书群"产出 N/5 篇 · 用时 X 分"
→ Loop 自己停

什么时候停？达到目标（5 篇通过） · 跑够轮数（≤15 轮防发散） · 超时（60 分钟）— 三选一。 自评只是 AI 的初筛，所有产出都要运营人最后点 approve / reject。

17 个工具 · 三组功能 · 老板看个概念

不用记名字 · 就记三类：读数据 / 干活 / 评估

读数据7 个

让 AI 知道现在有哪些问题、哪些模板、哪些文章、哪些账号
list_question_pools · list_question_items · list_prompt_templates · list_pipelines · list_articles · list_accounts · get_article

干活6 个

让 AI 真的产出：生文 · 配图 · 给文章打标 · 建分发任务 · 推飞书
compose_article · illustrate_article · submit_review_decision · set_review_status · create_distribute_task · notify_feishu

评估4 个

让 AI 自评 + 看历史表现，下一轮能跑得更准
score_recent_articles · get_template_performance · get_account_performance · record_publish_metrics

设计原则："一个工具一件事" — 不做万能 batch、不做单字段 setter。AI 看到工具名就知道用来干啥，不用读文档。
