# UI 升级设计文档 · 暗色玻璃科技风

- **日期**：2026-06-09
- **目标**：把 Geo 协作平台前端从现有「暖色编辑/纸感」风格，全面重构为**更现代、时尚、抓人眼球**的「暗色玻璃科技风」，同时保留长文写作的可用性。
- **状态**：已通过头脑风暴确认，待转实现计划。

## 1. 背景与现状

前端为 React 19 + Vite + TypeScript（strict）+ Tiptap，**无 Tailwind、无组件库**，全站样式集中在单文件 `web/src/styles.css`（约 2863 行）。该文件已经是 **CSS 变量驱动**（`--accent` / `--fg` / `--paper` / `--hair` 等）、类名语义清晰、按注释分区（SHELL / SIDEBAR / WORKSPACE / TOPBAR / BUTTONS / BADGES / INPUTS / 各 feature）。

现有视觉：暖奶油底（`#F5F2EB`）、Fraunces 衬线标题、靛蓝强调（`#5B5FE9`），整体安静、书卷气。

平台约 10 个功能页（导航见 `web/src/types.ts` 的 `navItems`）：智能体管理、AI 生文、内容管理、提示词管理、图片库、媒体矩阵、分发引擎、系统状态，外加登录页、用户管理、审计日志。

## 2. 已确认的方向决策

| 决策点 | 结论 |
|---|---|
| 整体风格 | **A · 暗色科技 / 玻璃拟态**（深色底 + 紫蓝霓虹渐变 + 玻璃质感卡片） |
| 暗色范围 | **暗壳 + 亮纸编辑器**：导航/列表/仪表盘走暗色；文章编辑器与长文阅读区切为亮色"纸面"护眼 |
| 改造范围 | **全面重构**（token + 壳层 + 所有共用组件 + 逐个功能页调优） |
| 落地方式 | **就地重构 `styles.css`**（保留类名，靠变量级联；不拆分模块、不引入 Tailwind） |
| 主色 | 靛蓝 `#6d6bf6` → 紫 `#a855f7` 霓虹渐变，次色青 `#38bdf8`（已高保真草图确认） |

## 3. 设计 Token 层（`:root` 重写）

**原则**：保留现有变量名让级联自动生效，只替换取值并新增暗色/玻璃/霓虹/亮纸 token。

### 3.1 暗色外壳 token
- 底色：`--bg #0a0c14`、`--bg-2 #0e1120`；背景叠加两道径向光晕（左上靛蓝 ~28%、右上紫 ~20%）营造氛围。
- 玻璃面：`--glass rgba(255,255,255,.045)`、`--glass-strong rgba(255,255,255,.07)`；描边 `--line rgba(255,255,255,.08)`、`--line-2 rgba(255,255,255,.12)`；玻璃容器用 `backdrop-filter: blur(14px)`。
- 霓虹强调：`--acc-1 #6d6bf6`、`--acc-2 #a855f7`、`--grad linear-gradient(120deg, var(--acc-1), var(--acc-2))`；次色 `--cyan #38bdf8`。
- 文字：`--t1 #eceefb`（主）、`--t2 #99a0bd`（次）、`--t3 #5f667e`（弱）。
- 状态色（暗底霓虹版，各配 ~14% 透明底 + ~30% 同色描边）：绿 `#34d399`、蓝 `#38bdf8`、琥珀 `#fbbf24`、红 `#f87171`。
- 现有别名（`--accent` / `--fg` / `--fg-2` / `--fg-3` / `--paper` / `--cream` / `--sidebar` / `--hair` 等）映射到上述新值，使旧类名无需改动即可换肤。

### 3.2 亮纸 scope token
一组 `--paper-*` 变量（纸底 `#fbf9f4`、墨字 `#1a1a1a`、次字、米色描边、衬线标题），**仅在 `.paper-scope` 容器内覆盖**对应的内容变量，实现局部翻亮。

## 4. 文件内分层结构

`styles.css` 顶部声明 `@layer tokens, base, shell, components, features;`（若与现有写法冲突则退化为注释分区，不强求 `@layer`）。重写顺序固定：**tokens → base → shell（侧边栏/顶栏）→ components（按钮/徽章/面板/卡片/输入/列表/状态）→ 各 feature 专项**。保持现有 `/* ═══ 分区 ═══ */` 注释风格。

## 5. 组件处理规范

- **侧边栏**：磨砂玻璃底；激活项靛蓝→紫渐变高亮 + 左侧发光竖条 + 右侧光点；底部用户卡玻璃化。
- **顶栏**：半透明 + 模糊；标题带副信息（统计/计数）；右侧渐变主按钮 + 渐变头像。
- **按钮**：主按钮渐变填充 + 发光阴影；次按钮玻璃描边；危险态红、幽灵态透明各一套；禁用沿用现有降透明度。
- **徽章/状态**：霓虹胶囊（透明底 + 同色描边 + 同色字），覆盖已发布/待审核/草稿/失败/进行中等状态。
- **面板/卡片**：玻璃面 + 细描边 + 圆角 13–14px；悬浮时微抬升（`translateY(-2px)` + 阴影）。
- **数据卡**：右上角渐变光斑装饰。
- **输入/下拉/文本域**：玻璃底；聚焦时霓虹描边 + 外发光。
- **列表行**：悬浮玻璃高亮；长列表行**不加 backdrop-filter**（性能）。
- **滚动条 / 选区 / 链接**：暗色霓虹化。

## 6. 亮纸编辑器机制

给 Tiptap 编辑器容器及长文阅读区根节点加 `.paper-scope` class。该容器内：
- 覆盖内容相关 CSS 变量为 `--paper-*`，底色翻为米白纸面、正文转墨色；
- 标题沿用 Fraunces 衬线，延续品牌基因；
- 形成"亮色明信片嵌在暗色背景中"的视觉对比。

这是**局部 scope**，不引入全站 light/dark 主题切换与持久化，避免复杂度。`.paper-scope` 之外的样式不受影响（防泄漏：亮纸规则一律以 `.paper-scope` 前缀限定）。

## 7. 动效

统一沿用现有 `--easing: cubic-bezier(0.22, 1, 0.36, 1)`。允许：悬浮抬升、激活项渐入、按钮/光点的轻微发光。克制为主，避免大面积持续动画影响性能与专注。

## 8. 推广顺序（逐页）

1. **Token 层 + base**（换肤地基）
2. **壳层**：侧边栏 + 顶栏 + 登录/加载态
3. **共用组件**：按钮 / 徽章 / 面板 / 卡片 / 输入 / 列表 / 状态（完成后全站观感即整体跃迁）
4. 内容管理 → 5. AI 生文 → 6. 智能体管理（pipelines）→ 7. 分发引擎/任务 → 8. 图片库 → 9. 账号/媒体矩阵 → 10. 系统状态/审计/用户管理 → 11. 登录页精修 → 12. 编辑器亮纸 scope

每步完成后用可视化伴侣或真机截图核对再进入下一步。

## 9. 风险与对策

| 风险 | 对策 |
|---|---|
| 暗底对比度 / 可读性 | 正文用 `--t1/--t2` 而非纯灰；关键文字满足 WCAG AA；状态色保证暗底可辨 |
| `backdrop-filter` 性能 | 限制模糊层数量；长列表行不加模糊；必要处用半透明纯色兜底 |
| 亮纸样式泄漏到暗壳 | 所有亮纸规则严格以 `.paper-scope` 前缀限定 |
| 第三方控件（Tiptap 工具栏等）配色不协调 | 在编辑器 scope 内单独覆盖工具栏类（头条字节设计系统的 `byte-*` 不在前端范围，无需处理） |
| 回归 | 纯 CSS 改动、DOM 结构不变、JSX 基本不动；分步提交，可逐步回滚 |

## 10. 范围与非目标

**范围内**：`web/src/styles.css` 重构；必要时为新增装饰元素（光斑、发光条、`.paper-scope` 包裹层）在对应 feature 的 `.tsx` 中加**少量纯展示性 className / 包裹 div**，不改业务逻辑、不改数据流、不改 API。

**非目标（YAGNI）**：
- 不做全站 light/dark 主题切换与用户偏好持久化（仅亮纸 scope）。
- 不拆分 CSS 为多文件、不引入 Tailwind 或任何 UI 组件库。
- 不改动后端、不改路由、不改组件的状态/交互逻辑。
- 不处理头条等平台站内自动化页面（`byte-*`，属浏览器自动化范围）。
