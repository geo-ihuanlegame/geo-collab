这张图传达的 3 个关键设计决策

  • 工作流引擎是内核（不是数据库、不是 LLM） — 所有子系统都是它的节点；这是 A 路线"工作流节点搭建"的精确落地
  • 归因层是独立 Layer 5，不嵌在数据回收里 — 痛点 1 要单独建模，做"分层归因"而不是单一指标
  • 视频在 Layer 3 内但标 P2，二期才接入；一期 Layer 3 只跑文/图

2.4 · 扫描频率

  • L1 品牌词：每日 3 次（早/午/晚），近实时
  • L2 类目词：每日 1 次（凌晨 2-4 点错峰）
  • L3 场景词：每 3 天 1 次
  • L4 长尾词：每周 1 次轮转扫
  • 触发式：新内容发布后 24h / 72h 触发针对性回扫，看是否被引用

2.5 · 输出消费者（数据回流）

  • → 工作流-选题节点：哪些关键词需补内容

  • → 归因层-L1 露出归因：基础数据
  • → 内容生产-审核节点：竞品同场分析

3.5 · 审核节点（关键）

  • L1 自动规则：敏感词 / 风险词 / 数据校验（不能虚构数据）
  • L2 事实校验：RAG 比对自建知识库
  • L3 竞品对比合规：避免不当贬低竞品（法务红线）
  • L4 人工抽检：高 stakes 内容 100%审；其他按 10-20% 抽样
  • 反馈回流：被打回稿子作为负样本 → prompt 优化

4.3 · 风控规避（关键）

频率控制：每账号每日发布上限，时间分散（非整点、避高峰）
内容差异化：同主题不同账号 → 不同改写版（标题/开头/示例图都换）
IP/设备隔离：每账号独立代理 + 浏览器指纹隔离
账号健康度：每周扫描互动率/异常提示，分级预警
平台规则适配：小红书 18 字标题、知乎话题、头条头图比例等

5.3 · 综合归因评分公式

// 每篇 GEO 内容的归因分数
attribution_score(content_c) =
  0.10 × L1_exposure_rate(c)
  + 0.30 × L2_utm_conversion(c)
  + 0.20 × L3_time_correlation(c)
  + 0.30 × L4_experiment_lift(c)
  + 0.10 × L5_user_recall(c)

// 输出："这篇内容带来 X 个下载，置信度 Y%"
{ downloads: ~127, confidence: 0.82 }
权重不是固定的 — 一期先用经验值，二期接入 ML 模型校准

5.4 · 关键工程组件

  • 专属落地页系统（L2 锚点）：每篇 GEO 内容自动生成一个 短链/landing.geo/xxxx 落地页，带 UTM 参数和来源识别
  • UTM 设计规范：utm_source=geo · utm_medium={platform} · utm_campaign={content_id} · utm_content={engine_target}
  • 渠道码 / 邀请码：落地页扫码或输入码下载，码与内容绑定
  • 实验调度器（L4 核心）：自动安排 A/B 关键词组、暂停周期、对照组隔离

6.5 · ROI 计算器（给老板看的"为什么投 GEO"）

// 月度 GEO 投入

  engineering_cost + content_production_cost (LLM API + 图片 + 视频)
  + distribution_cost (账号 + 风控 + 代理 + 平台费)
  + ops_human_cost (运营 + 审核)

// 月度 GEO 产出

  Σ(attributed_downloads × avg_LTV)
  + Σ(attributed_registers × avg_register_value)
  + brand_search_lift (品牌词搜索量提升的折现)

// 月度 ROI
ROI = (REVENUE - COST) / COST

{ monthly_roi: 1.8, payback_period_months: 4.2 }

7.4 · 视频 GEO 特征工程（视频时代的 GEO）

视频本身现在还不被 AI 直接"看"，AI 主要抓元数据（标题/描述/字幕）。2026 年起，多模态 AI（Gemini 多模态 / Sora 训练源）开始抓关键帧 + 字幕轨道。GEO 优化要兼顾两者。
标题 / 封面：实体明确（"XX App 怎么用"）+ 类目关键词 + 抓眼球开头
字幕轨道（关键）：完整 + 实体锚点 + 时间戳标记 — 这是 AI 训练的主要抓取点
描述：FAQ 块 + 引用源 + 章节标记（Chapter 0:00 - 0:30 引言 ...）
关键帧：清晰的品牌露出帧（Logo / UI 截图 / 产品名字幕条）

9.2 · 风险地图（6 大类 · 18 个识别项）

⚠ 严重度 = 出现概率 × 影响面 · S=Severe / H=High / M=Medium / L=Low
技术风险
工作流引擎选型错误
H
n8n / Dify 等是否支持 GEO 场景的定制需求（多账号池、复杂调度）？选错要换内核。
缓解：S0 必须有 POC 验证 3 个关键场景能跑
Playwright 反爬 / 验证码
S
国内平台对自动化敏感（小红书/知乎），账号封禁潮可能让分发瘫痪。
缓解：账号健康度监控 + 多账号轮转 + 浏览器指纹 + 国内 IP 代理池
AI 引擎抓取合规
M
部分 AI 引擎 ToS 禁止爬虫；监测层抓取存灰区风险。
缓解：优先用官方 API（智谱/Kimi 等有），Web 自动化频率严控
归因数据不全（D1 阻塞）
S
如果 App 没接归因 SDK，L2/L4 精度大跌，归因层只剩 L1+L3+L5。
缓解：本项目推动接入 AppsFlyer/Adjust；L5 用户自报作 fallback
业务风险
L4 实验暂停 GEO 投放
M
为了对照实验要暂停 1 周以上 GEO 投放，业务侧可能反对。
缓解：分平台/分关键词组实验，不全局暂停
视频 GEO hit rate 不达预期
H
2026 年视频 GEO 还在演进，多模态 AI 抓取视频内容能力有限。
缓解：先靠字幕轨道 GEO（AI 抓的核心），视频本身做品牌曝光
海外 MVP 运营能力不足
M
运营团队英文能力 / 文化把握 / 合规知识可能不够。
缓解：海外 MVP 阶段招 1 名海外运营，或外包初期内容
运营风险
账号封禁潮
S
多平台同步发同主题 → 算法识别 → 同周大范围封号。
缓解：D4.4 内容差异化 + 时间分散 + 账号健康度监控
平台政策变化
M
小红书改规则、知乎调算法 → 现有策略失效。
缓解：节点化架构便于策略热替换
法务 / 合规风险
AI 内容标记不一致
M
各平台政策不一（小红书强制标、头条不强制），漏标会被平台限流。
缓解：D3.4 平台政策表 + 分发节点自动适配
竞品对比法务红线
M
GEO 内容含竞品对比，存"不当贬低"风险。
缓解：审核节点加入"竞品对比合规校验"
用户数据归因隐私
L
L2/L5 涉及用户行为数据采集。
缓解：用户隐私协议覆盖；L5 问卷匿名化
组织风险
App 团队配合发版
H
D5.3 L5 用户自报需要 App 团队发版改动，可能排期受阻。
缓解：本周就对接 App 团队 Lead，争取 S3 之前发版
数据 BI 团队配合度
M
D6.4 业务数据要从 BI 来，BI 团队对项目优先级不一定认可。
缓解：S0 期间高层对齐；接入方式选定时拉取避免侵入
老板对 ROI 预期管理
M
早期 ROI 可能<1（投入大 / 归因还没建立），可能被砍预算。
缓解：S2 起就有"露出层 ROI"对外披露；M9 发布会提前对齐预期
未知风险（黑天鹅）
新 AI 引擎出现 / 模型升级
M
2026 后期可能冒出新 player；现有引擎模型升级导致内容失效。
缓解：节点化架构 + 监测层可扩展，预留时间响应
监管对 AIGC 政策收紧
M
国内对 AIGC 标记 / 算法备案的要求可能强化。
缓解：法务每月跟进政策；保留全人工审通道作 fallback
