# AI 生文功能需求文档

**需求优先级**：中等  
**预计上线**：待排期  
**所有者**：待指派

---

## 1. 功能概述

在 Geo 协作平台中新增 **AI 生文模块**，支持用户通过配置 Skill 和提示词，使用 AI 智能生成文章。

**核心价值**：
- 加速内容创作效率
- 支持个性化配置（Skill + 提示词）
- 灵活的交互方式（一键生成 vs 对话迭代）
- 生成文章自动归档到文章管理模块

---

## 2. 用户角色和权限

- **用户范围**：所有系统用户（User）
- **权限控制**：项目 User 账号自动具有生文权限，无需额外审批

---

## 3. 核心功能需求

### 3.1 Skill 管理

**定义**：Skill 是一套预配置的 AI 生文方案，包含：
- Skill 名称（e.g. "geo-article-v2"）
- 模型/API 源配置（目前不固定，可支持切换）
- Skill 版本管理

**当前生产 Skill**：
```
C:\Users\Downloads\geo-article-v2  （同事维护，生产在用）
```

**功能需求**：
- [ ] 后端支持加载和管理多个 Skill
- [ ] 前端提供 Skill 选择下拉菜单
- [ ] 支持上传/更新 Skill 配置（仅管理员）
- [ ] Skill 状态标记（启用/禁用）

### 3.2 提示词管理

**定义**：提示词（Prompt）是传递给 AI 的指令，控制文章生成风格和内容。

**特点**：
- 不固定，用户可动态调整
- 支持预设和临时创建

**功能需求**：
- [ ] 提前输入提示词并保存为**预设模板**
- [ ] 一键生成时可以：
  - 选择已保存的提示词模板
  - 现场输入新的提示词
- [ ] 提示词编辑界面（文本框，支持变量占位符）
- [ ] 提示词版本历史（可选）

### 3.3 一键生成

**流程**：
```
用户选择 Skill → 选择/输入提示词 → 点击"生成" → AI 响应 → 文章自动存储
```

**交互细节**：
1. 勾选启用的 Skill（如果只有一个，默认勾选）
2. 勾选预设提示词 **或** 在文本框输入新提示词
3. 点击"生成文章"按钮
4. 显示生成进度（进度条/加载动画）
5. 生成完成后：
   - 显示预览
   - 确认按钮 → 保存到文章管理
   - 编辑按钮 → 修改后再保存
   - 放弃按钮 → 回到生文页面

**UI 参考位置**：
- 独立页面（如 `/features/generation`）或侧边栏面板
- 或在文章管理页面增加"AI 生成"模块

### 3.4 Agent 对话式生成

**流程**：
```
用户发起对话 → Agent 提问（需求澄清）
→ 用户回答 → Agent 迭代优化
→ 用户确认出文 → 文章存储
```

**功能需求**：
- [ ] 对话界面（消息历史）
- [ ] Agent 逻辑：
  - 首轮：理解用户需求，提出澄清问题
  - 中轮：根据回答迭代提示词
  - 确认轮：生成最终文章，显示预览
- [ ] 支持改进和重新生成
- [ ] 对话历史保存（可选）

## 4. 文章流转

### 4.1 生成文章格式

- **优先格式**：对 AI 最友好的格式（Markdown > HTML > 其他）
- **存储格式**：Tiptap JSON（项目现有格式）
- **格式转换**：如果 AI 输出不是 Tiptap JSON，需要做格式清洗转换

### 4.2 文章存储

**生成的文章**：
- 自动创建为**草稿**（status = "draft"）
- 存储到文章管理模块
- 用户可以在"文章管理"页面查看、编辑、发布

**元数据**：
- 生成时间（created_at）
- 使用的 Skill 名称
- 使用的提示词（记录供审计）
- 生成来源标记（"AI Generated"）

## 5. 技术架构

### 5.1 后端

**新增 API 端点**：
```
POST /api/generation/generate
  - payload: { skill_id, prompt, mode }
  - response: { article_id, preview, status }

POST /api/generation/generate-with-agent
  - payload: { message, conversation_id }
  - response: { agent_message, conversation_id }

GET /api/generation/skills
  - response: { skills: [...] }

GET /api/generation/prompts/templates
  - response: { templates: [...] }

POST /api/generation/prompts/templates
  - payload: { name, content }
  - response: { template_id }
```

**新增模块**：
```
server/app/modules/generation/
├── __init__.py
├── generation_service.py    # 核心生文逻辑
├── skill_manager.py         # Skill 加载和管理
├── prompt_template.py       # 提示词模板管理
├── agent_handler.py         # Agent 对话处理
└── article_formatter.py     # 格式转换
```

**配置**：
- `GEO_GENERATION_SKILLS_PATH` = Skill 目录路径
- `GEO_GENERATION_ENABLED` = 功能开关

### 5.2 前端

**新增页面/组件**：
```
web/src/features/generation/
├── GenerationPage.tsx       # 生成页面（单独路由）
├── SkillSelector.tsx        # Skill 选择器
├── PromptEditor.tsx         # 提示词编辑器
├── OneClickGeneration.tsx    # 一键生成模块
├── AgentChat.tsx            # Agent 对话模块
└── PreviewModal.tsx         # 生成预览确认
```

**UI 位置**：
- 单独页面：`/generation` 或 `/content-generation`
- 在导航菜单中增加"内容生成"入口

## 6. 可配置和扩展性

### 6.1 Skill 扩展

- 支持多个 Skill 同时存在
- **Skill 配置格式**：保留原有格式（`geo-article-v2` 现有格式）
- **新增导入**：支持导入新 Skill 时自动转换为 AI 友好的格式
- 可动态加载新 Skill 而不需重启

### 6.2 提示词模板库

- 用户可创建和保存提示词模板
- 管理员可创建全局模板供所有用户使用
- 支持提示词变量（e.g. `{{article_type}}`, `{{tone}}`）

### 6.3 AI 源切换

- 目前不关键，但架构需要支持
- 未来可支持：Claude API → GPT → 本地 LLM

## 7. 数据模型

### 新增表

```sql
-- Skill 配置
CREATE TABLE generation_skill (
  id INT PRIMARY KEY AUTO_INCREMENT,
  name VARCHAR(255) NOT NULL,
  version VARCHAR(50),
  config_path VARCHAR(500),
  is_enabled BOOLEAN DEFAULT TRUE,
  created_at DATETIME,
  updated_at DATETIME
);

-- 提示词模板
CREATE TABLE prompt_template (
  id INT PRIMARY KEY AUTO_INCREMENT,
  user_id INT,                    -- 用户模板（NULL = 全局模板）
  name VARCHAR(255) NOT NULL,
  content TEXT NOT NULL,
  variables JSON,                 -- 模板变量列表
  is_public BOOLEAN DEFAULT FALSE,
  created_at DATETIME,
  updated_at DATETIME
);

-- 生成记录（审计）
CREATE TABLE generation_record (
  id INT PRIMARY KEY AUTO_INCREMENT,
  article_id INT NOT NULL,
  user_id INT NOT NULL,
  skill_id INT,
  prompt_used TEXT,               -- 使用的完整提示词
  mode ENUM('one_click', 'agent'), -- 生成方式
  status VARCHAR(50),             -- 成功/失败
  error_message TEXT,
  conversation_id VARCHAR(100),   -- Agent 对话 ID（如适用）
  generated_at DATETIME,
  FOREIGN KEY (article_id) REFERENCES article(id)
);
```

## 8. 非功能需求

- **性能**：生成响应时间 < 30s（取决于 AI 服务）
- **并发**：支持多用户同时生成
  - FastAPI 异步处理，独立并发无需队列
  - 直接调用外部 AI API（API 本身支持并发）
  - 每个请求独立的数据库 Session，并发保存安全
- **容错**：生成失败时提示用户并保存错误日志
- **审计**：记录所有生成行为（用户、Skill、提示词、时间）
- **成本控制**：无生成次数限制
