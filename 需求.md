# AI 生文助手 - 功能需求文档

## 1. 概述

AI 生文助手是一个集成在 Geo 协作平台内的智能内容生成模块，提供两种内容生成模式：
- **一键生成**：用户选择预设技能和模板，一次性生成完整文章
- **Agent 对话**：用户与 AI 进行多轮对话，逐步完善内容需求，最后生成文章

## 2. 用户角色与权限

### 角色定义
- **内容操作者（Operator）**：可使用一键生成和 Agent 对话功能
- **管理员（Admin）**：可以管理技能库和提示词模板库，拥有所有内容操作权限

### 权限模型
- 一键生成和 Agent 对话功能对 Operator 及以上开放
- 技能和模板管理功能仅对 Admin 开放
- 不限制（所有已验证用户都可访问）

## 3. 核心功能模块

### 3.1 技能管理（Skill Management）
- **功能描述**：管理 AI 生成文章时的预设技能库
- **UI 交互**：
  - 显示技能列表（名称、描述、使用次数、创建时间）
  - 创建新技能（编辑器输入技能定义、验证输入）
  - 编辑现有技能
  - 删除技能
  - 技能搜索和过滤

### 3.2 提示词管理（Prompt Management）
- **功能描述**：维护提示词模板库，支持参数化模板
- **UI 交互**：
  - 按技能分类显示提示词模板
  - 创建新模板（名称、描述、模板内容、关联技能）
  - 编辑现有模板
  - 删除模板
  - 模板预览和参数检查

### 3.3 一键生成（One-Click Generation）
- **功能描述**：用户快速生成文章内容
- **操作流程**：
  1. 选择技能（下拉菜单/卡片选择）
  2. 选择提示词模板
  3. 输入自定义提示词或覆盖模板参数
  4. 预览生成结果
  5. 确认并保存为草稿文章或直接发布
- **生成结果**：
  - 显示生成的文章标题、内容、元数据
  - 支持编辑生成的内容
  - 保存为新文章或合并到现有文章

### 3.4 Agent 对话（Agent Dialogue）
- **功能描述**：多轮交互式内容生成
- **操作流程**：
  1. 用户描述内容需求（自然语言）
  2. Agent 理解需求并提出澄清问题或反馈
  3. 用户继续对话，补充或修改需求
  4. Agent 基于对话历史生成文章
  5. 用户确认并保存
- **对话管理**：
  - 显示对话历史（用户消息、Agent 响应）
  - 支持对话撤销/重试
  - 在 3 条对话以上自动生成草稿
  - 对话中实时展示生成中的文章预览

## 4. 核心架构设计

### 4.1 后端架构
- **AI 集成层**：对接 LLM 服务（如 Claude、GPT-4 等）
- **提示词处理**：模板解析、参数替换、验证
- **对话管理**：存储对话历史、维护上下文
- **文章生成**：使用标准 ArticleCreatePayload，集成现有文章保存流程

### 4.2 前端架构
- **生成页面**：两种模式切换（一键/Agent）
- **技能选择器**：下拉/卡片组件
- **提示词编辑器**：支持模板参数高亮
- **对话面板**：消息列表、输入框、生成预览
- **结果展示**：Markdown 渲染、编辑器集成

### 4.3 数据流
```
用户输入 → 前端收集 → API 调用 → 后端处理 → LLM 调用 → 结果处理 → 前端展示 → 文章保存
```

## 5. 数据模型

### 5.1 Skill（技能）
```python
{
  "id": int,
  "name": str,
  "description": str,
  "definition": str,  # 技能定义（JSON/纯文本）
  "usage_count": int,
  "created_at": datetime,
  "updated_at": datetime,
  "created_by": int  # 管理员 ID
}
```

### 5.2 PromptTemplate（提示词模板）
```python
{
  "id": int,
  "skill_id": int,
  "name": str,
  "description": str,
  "content": str,  # 模板内容，支持 {param} 格式参数
  "parameters": list,  # ["param1", "param2"]
  "created_at": datetime,
  "updated_at": datetime
}
```

### 5.3 GenerationTask（生成任务）
```python
{
  "id": int,
  "article_id": int | null,
  "mode": "one_click" | "agent_dialogue",
  "skill_id": int | null,
  "prompt": str,
  "status": "pending" | "generating" | "completed" | "failed",
  "result": str,  # 生成的文章内容（JSON 或 Markdown）
  "created_at": datetime,
  "updated_at": datetime
}
```

### 5.4 ConversationMessage（对话消息）
```python
{
  "id": int,
  "task_id": int,
  "role": "user" | "assistant",
  "content": str,
  "created_at": datetime
}
```

## 6. API 端点设计

### 6.1 技能管理
- `GET /api/skills` - 获取技能列表
- `POST /api/skills` - 创建技能
- `PUT /api/skills/{id}` - 编辑技能
- `DELETE /api/skills/{id}` - 删除技能

### 6.2 提示词模板
- `GET /api/prompt-templates` - 获取模板列表
- `GET /api/prompt-templates?skill_id={id}` - 按技能筛选
- `POST /api/prompt-templates` - 创建模板
- `PUT /api/prompt-templates/{id}` - 编辑模板
- `DELETE /api/prompt-templates/{id}` - 删除模板

### 6.3 一键生成
- `POST /api/generation/one-click` - 执行一键生成
  - 请求体：`{ "skill_id": int, "template_id": int, "prompt_override": str }`
  - 响应体：`{ "task_id": int, "article": Article }`

### 6.4 Agent 对话
- `POST /api/generation/dialogue/start` - 开始对话
  - 响应体：`{ "task_id": int }`
- `POST /api/generation/dialogue/{task_id}/message` - 发送对话消息
  - 请求体：`{ "content": str }`
  - 响应体：`{ "message_id": int, "response": str }`
- `GET /api/generation/dialogue/{task_id}/messages` - 获取对话历史
- `POST /api/generation/dialogue/{task_id}/generate` - 生成最终文章
  - 响应体：`{ "article": Article }`

## 7. 非功能需求

### 7.1 性能
- 一键生成响应时间 < 5秒
- Agent 对话首条响应 < 3秒
- 技能/模板列表加载 < 1秒

### 7.2 可靠性
- 生成任务支持断点续传
- 对话历史持久化存储
- 生成失败时提供明确错误提示

### 7.3 用户体验
- 生成过程实时反馈进度
- 对话结果预览支持编辑后直接保存
- 技能和模板支持搜索和快速过滤

### 7.4 安全性
- 权限检查：仅 Admin 可管理技能/模板
- 内容审核：保存文章前进行内容检查
- 审计日志：记录所有操作（生成、编辑、删除）
